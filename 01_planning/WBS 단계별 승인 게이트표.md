# WBS 단계별 승인 게이트표

## 1. 목적

DBT/Airflow 전환 WBS의 각 단계에서 고객 협의와 승인이 필요한 항목을 정리하여, 일정 지연과 범위 변경 리스크를 줄인다.

## 2. 승인 게이트 원칙

| 원칙 | 설명 |
|---|---|
| 사전 협의 | 단계 시작 전에 핵심 범위를 먼저 합의한다. |
| 종료 승인 | 단계 종료 시점에 승인 대상 산출물을 확보한다. |
| 이슈 분리 | 승인 지연 가능 항목은 별도 이슈로 관리한다. |
| 기준 명확화 | 검증 기준과 오픈 기준은 문서로 고정한다. |

## 3. 단계별 승인 게이트

### 3.1 분석 단계

| 구분 | 항목 | 고객 협의 포인트 | 승인 필요 여부 |
|---|---|---|---|
| 협의 | 전환 대상 테이블 목록 | 포함/제외 기준, 우선순위 확인 | 필요 |
| 협의 | 전환 대상 JOB 목록 | 대상 범위와 분류 기준 확인 | 필요 |
| 협의 | AS-IS 현황 | 예외 JOB, 수작업 처리 여부 확인 | 필요 |
| 협의 | 개발환경 표준 | Snowflake 접속, Git, 배포 방식 안내 | 필요 |
| 협의 | Airflow 개발환경 표준 | Airflow, Cosmos, Connection, Variable, Secret 구성 안내 | 필요 |
| 승인 | 요구사항정의서 | 범위 및 목표 확정 | 필요 |
| 승인 | Airflow 개발환경 구성표 | 개발/실행 환경 표준 승인 | 필요 |
| 승인 | 분석 결과 및 전환 범위 | 1차/2차 오픈 포함 범위 확정 | 필요 |

|---|---|---|
| 분석 | 범위 정의 | 요구사항정의서, 전환 범위, Airflow 개발환경 구성표 |
| 설계 | 기준 정의 | 검증 절차서, dbt 구조, Airflow 아키텍처, AI 적용 전략 |

## 5. 운영 원칙

- 고객 협의 항목은 단계 시작 시점에 먼저 확정한다.
- 승인 항목은 단계 종료 전 반드시 확보한다.
- 승인 지연 가능성이 있는 항목은 별도 이슈로 관리한다.
- AI 적용 범위와 검증 기준은 설계 단계에서 고객과 합의한다.
