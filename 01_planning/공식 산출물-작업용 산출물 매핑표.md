# 공식 산출물-작업용 산출물 매핑표

## 1. 목적

작업용 산출물이 어떤 공식 산출물로 귀결되는지 문서 단위로 정의하여, 초안과 승인본의 연결을 명확히 한다.

## 2. 매핑 원칙

| 원칙 | 설명 |
|---|---|
| 1:1 우선 | 작업용 산출물 1개는 원칙적으로 공식 산출물 1개에 연결한다. |
| 보조 매핑 허용 | 필요한 경우 보조 공식 산출물 1개까지 연결할 수 있다. |
| 전환 기준 명시 | 어떤 조건에서 공식 산출물로 전환되는지 기록한다. |
| 승인자 명확화 | 최종 승격 책임자를 남긴다. |

## 3. 매핑 표

| 작업용 산출물명 | 연결 공식 산출물명 | 매핑 목적 | 전환 시점 | 책임자 | 승인자 | 비고 |
|---|---|---|---|---|---|---|
| 분석 메모 | 요구사항정의서 | 현황과 판단 근거를 요구사항으로 승격 | 분석 종료 시 | 박효진 | 김무경 | 범위 확정 전 초안 |
| 초안 설계서 | dbt 모델 정의서 | 설계안을 공식 구조로 고정 | 설계 리뷰 시 | 김진형 | 김무경 | 모델 표준 반영 |
| 검증 체크리스트 | 전환 검증 절차서 | 검증 항목을 절차 문서로 정리 | 테스트 준비 시 | 김진형 | 박효진 | 기준 명확화 |
| 실험 로그 | 데이터 검증 결과서 | 실험 결과를 검증 결과로 정리 | 검증 완료 시 | 김재웅 | 김무경 | 재현성 확보 |
| 회의 후속 정리본 | 회의록 / 액션아이템표 | 회의 내용을 실행 문서로 전환 | 회의 직후 | 최경연 | 박효진 | 담당/기한 포함 |
| 이슈 메모 | 이슈 리스트 | 미해결 항목을 추적 문서로 연결 | 이슈 등록 시 | 박효진 | 김무경 | 에스컬레이션 대상 |

## 4. 운영 기준

- 작업용 산출물은 공식 산출물의 초안 또는 보조 근거로만 사용한다.
- 공식 산출물로 승격되면 문서명과 버전을 명확히 구분한다.
- 매핑 변경은 승인자 확인 후 반영한다.
