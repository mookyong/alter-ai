## 현실적인 진행 전략

이 규모는 **수작업 전환 중심으로 가면 일정이 매우 위험**합니다.
따라서 초반 2개월 안에 반드시 아래 체계를 만들어야 합니다.

1. **DataStage JOB 자동 분석**

   * SQL 추출
   * 소스/타겟 테이블 추출
   * 선후행 관계 추출
   * 변환 유형 분류

2. **전환 유형 분류**

   * 단순 SQL 전환
   * JOIN/집계 중심 전환
   * Lookup/Transformer 포함 전환
   * 복잡 Sequence 전환
   * 수작업 검토 필요 JOB

3. **dbt/Airflow 템플릿 기반 대량 전환**

   * dbt 모델 템플릿
   * macro 템플릿
   * DAG 템플릿
   * 검증 SQL 템플릿
   * 로그/모니터링 템플릿

4. **1차 오픈 대상 우선 개발**

   * 편의점/수퍼/MD 관련 공통 테이블부터 먼저 전환
   * 그다음 도메인별 마트/업무 로직 전환
   * 오픈 3주 전부터는 신규 개발보다 결함 수정/검증 중심으로 전환
