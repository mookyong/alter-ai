## About

**alter-ai**는 개인의 사고 방식, 개발 스타일, 문제 해결 흐름을 AI 에이전트 구조로 구현하기 위한 개인 연구 프로젝트입니다.

단순한 챗봇이나 자동화 도구를 만드는 것이 아니라,  
개발자가 문제를 바라보고, 분석하고, 판단하고, 실행하는 과정을 구조화하여  
**나를 닮은 디지털 페르소나**, 즉 **AI alter-ego**를 만드는 것을 목표로 합니다.

이 프로젝트는 다음과 같은 질문에서 출발합니다.

> “AI가 단순히 명령을 수행하는 도구를 넘어,  
> 나의 사고 방식과 엔지니어링 철학을 반영한 협업자가 될 수 있을까?”

## Research Areas

### 1. Persona Modeling

개발자의 사고 방식, 코드 작성 기준, 문제 해결 패턴을 분석하여  
AI 에이전트의 역할과 행동 규칙으로 정의합니다.

예를 들어 다음과 같은 요소를 모델링합니다.

- 문제를 분석하는 방식
- 설계를 결정하는 기준
- 코드 품질을 판단하는 기준
- 오류를 추적하고 검증하는 과정
- 문서화와 설명 방식

---

### 2. Legacy Logic Transpilation

오래된 시스템, 문서, SQL, ETL 로직, 규칙 기반 코드를 분석하여  
현대적인 데이터 파이프라인, API, 에이전트 워크플로우로 전환하는 방법을 연구합니다.

주요 관심사는 다음과 같습니다.

- Legacy logic parsing
- Rule-based transformation
- SQL / ETL workflow interpretation
- AI-assisted refactoring
- Code generation with validation

---

### 3. Multi-Agent Orchestration

하나의 거대한 AI가 모든 일을 처리하는 방식이 아니라,  
역할이 분리된 여러 에이전트가 협업하는 구조를 지향합니다.

예시는 다음과 같습니다.

- Planner Agent: 요구사항 분석 및 작업 계획
- Architect Agent: 시스템 구조 설계
- Developer Agent: 코드 작성
- Reviewer Agent: 품질 검토
- Validator Agent: 테스트 및 검증
- Document Agent: 문서화

이 구조를 통해 복잡한 개발 작업을 더 안전하고 검증 가능한 방식으로 수행하는 것을 목표로 합니다.

## Engineering Principles

alter-ai는 다음과 같은 원칙을 중요하게 생각합니다.

### Logic Over Hype

AI 기술의 유행보다 중요한 것은  
명확한 구조, 검증 가능한 로직, 유지보수 가능한 설계라고 믿습니다.

### Human-in-the-Loop

AI가 모든 것을 자동으로 결정하는 것이 아니라,  
사람의 판단과 검증이 함께 포함된 구조를 지향합니다.

### Structured Autonomy

자율성은 무제한의 자유가 아니라,  
명확한 역할, 규칙, 검증 절차 안에서 더 강력해진다고 생각합니다.

### Build, Measure, Improve

아이디어를 선언하는 데서 끝나지 않고,  
작게 만들고, 측정하고, 개선하는 방식으로 발전시킵니다.

## Project Direction

현재 alter-ai는 다음과 같은 방향으로 확장될 예정입니다.

- `alter-ai-core`  
  AI 페르소나와 에이전트 실행 구조의 핵심 모듈

- `alter-ai-agents`  
  역할 기반 에이전트 정의 및 오케스트레이션 실험

- `alter-ai-transpiler`  
  레거시 로직과 문서를 현대적 워크플로우로 변환하는 엔진

- `alter-ai-playground`  
  실험용 데모, 프로토타입, 테스트 케이스 모음

> 각 저장소는 실험 단계에 따라 점진적으로 공개 및 정리될 예정입니다.
