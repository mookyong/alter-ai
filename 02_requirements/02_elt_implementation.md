# ELT 구현

## 목적

DataStage 기반 ETL을 dbt와 Airflow 중심의 ELT 구조로 전환한다.

## 핵심 요구

- 기존 DW(DB2)에서 데이터를 추출하여 AWS S3에 적재한다.
- S3 적재 데이터를 TO-BE DW로 이관하는 초기 이행 방식을 구현한다.
- IBM DataStage 기반 ETL 파이프라인을 dbt와 Airflow로 전환한다.
- 초기 이행 이후 일/월 단위 증분 적재가 원활히 수행되어야 한다.
- 위 이전 범위에 해당하는 Batch Job을 클라우드 환경에서 재구현한다.
- 위 이전 범위에 해당하는 모든 Staging 영역을 구현한다.

## 운영 관점

- 초기 이행과 증분 적재 모두를 안정적으로 운영할 수 있어야 한다.
- 데이터 정합성 검증이 가능한 구조를 고려한다.
