# 에이전트 루프 처리 정리

## 목적

특정 폴더에 모아둔 DB2 SQL 파일을 Snowflake SQL로 일괄 변환하기 위한 운영 방식을 정리한다.

핵심은 파일을 한 번에 모두 LLM에 넣는 것이 아니라, **파일 1개씩 순차 처리하는 루프 방식**으로 변환하는 것이다.

## 전제

- 원본 DB: `DB2`
- 대상 DB: `Snowflake`
- 변환 범위: `DML`
- 예문 파일 위치: `sql/`
- 변환 결과 위치: `output/`
- 실패 파일 위치: `failed/`

## 순수 LLM과 에이전트의 차이

### 순수 LLM

- 폴더를 스스로 읽을 수 없다.
- 파일 시스템을 순회할 수 없다.
- 루프를 직접 돌릴 수 없다.
- 저장도 할 수 없다.

즉, 순수 LLM만으로는 `폴더 전체 -> 자동 반복 변환`이 불가능하다.

### opencode 에이전트 + 도구

- `glob`으로 파일 목록 수집 가능
- `read`로 파일 1개씩 읽기 가능
- `bash`로 검증 가능
- `edit` / `apply_patch`로 결과 저장 가능
- 실패 파일만 재처리 가능

즉, **opencode 내부 에이전트가 도구 루프를 관리**하면 파일 단위 자동 변환이 가능하다.

## 전체 흐름

1. `sql/` 아래 `.sql` 파일을 수집한다.
2. 파일을 1개씩 읽는다.
3. 읽은 파일 내용만 LLM에 전달한다.
4. DB2 SQL을 Snowflake SQL로 변환한다.
5. 검증 규칙은 SQL 주석으로 남긴다.
6. 성공 결과는 `output/`에 저장한다.
7. 실패한 파일은 `failed/`에 남긴다.
8. 재시도 시 실패 원인을 함께 넣어 다시 변환한다.
9. 최종적으로 성공/실패 요약을 저장한다.

## 변환 규칙

- 파일 1개씩만 처리한다.
- 의미 보존을 우선한다.
- Snowflake에 직접 대응되지 않는 DB2 전용 문법은 제거하거나 대체하고, 주석으로 근거를 남긴다.
- 출력은 SQL만 허용한다.
- 설명문은 SQL 주석 외에는 넣지 않는다.
- 검증 규칙은 `-- Validation rules:` 아래에 적는다.

## 실패 재시도 규칙

### 재시도 기준

- 첫 실패 시 1회 재시도한다.
- 재시도 시 이전 실패 원인을 함께 사용한다.
- 같은 실패를 반복하지 않도록 한다.

### 실패 유형

- `token_limit`
- `syntax_error`
- `semantic_unclear`
- `empty_response`
- `validation_failed`

### 실패 후 처리

- 2회 실패하면 `failed/`에 보관한다.
- 필요하면 `failed/` 전용 재처리 모드로 다시 시도한다.
- 성공한 실패본은 옵션에 따라 정리할 수 있다.

## 로그 포맷

### 실패 로그 예시

```json
{
  "file": "sql/db2_04_paging_rownum.sql",
  "status": "failed",
  "attempt": 2,
  "error_type": "syntax_error",
  "message": "QUALIFY clause rejected",
  "source_db": "db2",
  "target_db": "snowflake",
  "stage": "llm_convert",
  "timestamp": "2026-05-22T12:34:56Z"
}
```

### 성공 로그 예시

```json
{
  "file": "sql/db2_01_skip_locked.sql",
  "status": "success",
  "attempt": 1,
  "source_db": "db2",
  "target_db": "snowflake",
  "stage": "save",
  "output_file": "output/db2_01_skip_locked.sql",
  "timestamp": "2026-05-22T12:34:56Z"
}
```

## 프롬프트 파일

루트의 `db2_to_snowflake_prompt.md`를 단일 원문(source of truth)으로 사용한다.

포함 내용:

- 운영용 최종 프롬프트
- 파일 1개씩 루프 처리 버전
- 실패 로그 포맷
- 권장 실패 유형

## opencode 내부 구성

### subagent

- 파일: `.opencode/agent/db2-snowflake-converter.md`
- 이름: `db2-snowflake-converter`
- 모드: `subagent`

역할:

- `sql/`의 파일을 하나씩 읽는다.
- `db2_to_snowflake_prompt.md`를 기준으로 변환한다.
- 결과를 `output/`에 저장한다.
- 실패 시 `failed/`에 남긴다.

### opencode.json

루트의 `opencode.json`에는 다음 command를 추가했다.

- `db2-snowflake-convert`
- `db2-snowflake-retry-failed`

이 command들은 `db2-snowflake-converter` subagent를 호출하는 용도다.

## 생성된 파일들

- `db2_to_snowflake_prompt.md`
- `convert_db2_sql_folder.py`
- `.opencode/agent/db2-snowflake-converter.md`
- `opencode.json`
- `sql/` 원본 예문
- `output/` 변환본

## 주의사항

- opencode 설정/에이전트 파일은 시작 시 로드되므로, 변경 후에는 opencode를 종료하고 재시작해야 반영된다.
- 루트 `agent/` 폴더를 임의로 두고 `opencode.json`에서 직접 참조하는 방식보다는, opencode가 지원하는 `.opencode/agent` 구조를 따르는 것이 안전하다.
- 유지보수용 상세 규칙은 `db2_to_snowflake_prompt.md`에 모으고, 에이전트 파일은 얇게 유지하는 것이 좋다.
