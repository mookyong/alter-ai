# 산출물 정의표

## 1. 목적

프로젝트에서 생산되는 모든 문서를 공식 산출물, 작업용 산출물, 참고용 산출물로 구분하여 혼선을 줄인다.

## 2. 분류 기준

| 구분 | 설명 |
|---|---|
| 공식 산출물 | 고객 협의/승인, 단계 종료, 대외 공유에 사용하는 문서 |
| 작업용 산출물 | 내부 검토, 초안 작성, 실험, 정리, 체크를 위한 문서 |
| 참고용 산출물 | 직접 승인 대상은 아니지만 판단 근거로 사용하는 문서 |

## 3. 관리 원칙

| 원칙 | 설명 |
|---|---|
| 문서 단위 관리 | 산출물은 문서 단위로 정의한다. |
| 목적 명시 | 각 문서의 사용 목적을 먼저 고정한다. |
| 승격 가능성 | 작업용 산출물은 공식 산출물로 승격될 수 있다. |
| 버전 관리 | 초안과 확정본의 구분을 유지한다. |

## 4. 공통 속성

| 항목 | 설명 |
|---|---|
| 산출물명 | 문서의 공식 명칭 |
| 구분 | 공식 / 작업용 / 참고용 |
| 목적 | 생성 이유와 사용 목적 |
| 작성 주체 | 초안을 작성하는 담당자 |
| 검토 주체 | 내용 검토 담당자 |
| 승인 주체 | 최종 승인 담당자 |
| 저장 위치 | 보관 경로 |
| 공개 범위 | 내부 전용/고객 공유/대외 공유 |
| 승격 기준 | 작업용에서 공식 산출물로 전환되는 조건 |
| 보관/폐기 기준 | 유지 여부 판단 기준 |

## 5. 산출물 예시

| 산출물명 | 구분 | 목적 | 비고 |
|---|---|---|---|
| 요구사항정의서 | 공식 | 전환 범위와 목표 확정 | 고객 승인 대상 |
| dbt 모델 정의서 | 공식 | 설계 기준 고정 | 고객/내부 공유 |
| 전환 검증 절차서 | 공식 | 검증 방식과 책임 정리 | 테스트 기준 문서 |
| 데이터 검증 결과서 | 공식 | 정합성 결과 보고 | 단계 종료 근거 |
| 분석 메모 | 작업용 | 현황 정리 및 판단 재료 | 초안 문서 |
| 초안 설계서 | 작업용 | 설계안 검토 | 공식 문서 전 단계 |
| 검증 체크리스트 | 작업용 | 검증 항목 확인 | 결과서 입력용 |
| 실험 로그 | 작업용 | 시도 결과 기록 | 재현성 확보 |
| 회의 후속 정리본 | 작업용 | 회의 결과 정리 | 액션아이템 원천 |
| 이슈 메모 | 참고용 | 판단 근거 정리 | 필요 시 승격 |
