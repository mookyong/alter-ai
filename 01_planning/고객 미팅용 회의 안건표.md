# 고객 미팅용 회의 안건표

## 1. 목적

DBT/Airflow 전환 WBS 진행 시 고객과 사전에 협의하거나 승인받아야 할 항목을 회의 안건 형태로 정리한다.

## 2. 회의 운영 원칙

| 원칙 | 설명 |
|---|---|
| 사전 공유 | 회의 전 안건과 참고자료를 먼저 공유한다. |
| 결정 중심 | 협의만 하지 말고, 반드시 승인 또는 보류 결정을 남긴다. |
| 단계 연계 | 분석, 설계, 개발, 테스트, 이관 단계별로 안건을 구분한다. |
| 이슈 기록 | 결정 불가 항목은 별도 이슈로 남긴다. |

## 3. 단계별 회의 안건

### 3.1 분석 단계

| 안건 | 협의 포인트 | 고객 결정 필요 사항 |
|---|---|---|
| 전환 대상 테이블 목록 | 포함/제외 기준, 우선순위 | 전환 대상 범위 확정 |
| 전환 대상 JOB 목록 | 대상 분류, 예외 JOB 여부 | 1차 검토 대상 확정 |
| AS-IS 현황 | 현행 구조, 수작업 처리 여부 | 전환 필요 범위 확정 |
| 개발환경 표준 | Snowflake 접속, Git, 배포 방식 | 표준안 수용 여부 |
| Airflow 개발환경 표준 | Airflow, Cosmos, Connection, Variable, Secret | 표준안 수용 여부 |
| 요구사항 정의 | 전환 목표, 고객 기대사항 | 요구사항정의서 승인 |

### 3.2 설계 단계

| 안건 | 협의 포인트 | 고객 결정 필요 사항 |
|---|---|---|
| DDL/DML 전환 기준 | DB2 -> Snowflake 전환 원칙 | 전환 기준 승인 |
| dbt 모델 구조 | 네이밍, 폴더, macro 사용 기준 | 구조 표준 승인 |
| Airflow 아키텍처 | DAG 분리, TaskGroup, retry 정책 | 아키텍처 승인 |
| AI Agent 적용 방식 | 변환/검증 자동화 범위 | 적용 범위 승인 |
| 검증 기준 | row count, key integrity, column sample, aggregation consistency | 검증 기준 승인 |
| 오류 로그 체계 | JSON 로그 형식, 재수정 흐름 | 로그 표준 수용 여부 |

### 3.3 개발 단계

| 안건 | 협의 포인트 | 고객 결정 필요 사항 |
|---|---|---|
| 1차 오픈 대상 범위 | 공통/단순 모델 범위 | 1차 오픈 범위 승인 |
| 복합 변환 대상 | JOIN, 집계, Lookup 전환 방식 | 복합 모델 적용 승인 |
| Airflow 실행 범위 | DAG, TaskGroup, selector 적용 방식 | DAG 구현 범위 승인 |
| 예외 JOB 처리 | 수작업 유지, 후순위 전환 | 제외/보류 대상 승인 |
| 단위 테스트 기준 | 합격 기준, 재작업 조건 | 개발 완료 기준 승인 |
| AI 재수정 범위 | SQL/model/macro 수정 범위 | 재수정 허용 범위 합의 |

### 3.4 테스트 단계

| 안건 | 협의 포인트 | 고객 결정 필요 사항 |
|---|---|---|
| 통합 테스트 시나리오 | 리허설, 비교 검증, 재처리 흐름 | 테스트 시나리오 승인 |
| 데이터 정합성 기준 | 소스-타겟 비교, 허용 오차 | 정합성 기준 승인 |
| Airflow 통합 테스트 시나리오 | DAG 순서, 재처리, 알림 흐름 | DAG 테스트 시나리오 승인 |
| 결함 등급 기준 | 치명/중요/경미 분류 | 결함 처리 기준 승인 |
| 재처리 절차 | 실패 JOB 재처리, 재검증 방식 | 재처리 절차 승인 |
| Go/No-Go 기준 | 오픈 판단 조건 | 오픈 여부 결정 기준 승인 |

### 3.5 이관 단계

| 안건 | 협의 포인트 | 고객 결정 필요 사항 |
|---|---|---|
| 운영 이관 절차 | 운영팀 전달 방식, 책임 범위 | 이관 방식 승인 |
| 운영 가이드 | dbt/Airflow 운영 절차, 장애 대응 | 운영 문서 승인 |
| 교육 자료 | 운영자 교육 범위, 일정 | 교육 범위 승인 |
| 잔여 이슈 이관 | 미해결 이슈 처리 주체 | 후속 처리 방식 승인 |
| 종료 기준 | 프로젝트 종료 조건, 검수 항목 | 종료 승인 |
| Airflow 운영 이관 문서 | DAG 운영 인수인계 문서 | 운영 문서 승인 |

## 4. 회의 안건 요약

| 단계 | 핵심 안건 | 최종 결정 포인트 |
|---|---|---|
| 분석 | 범위, 요구사항, 현황, Airflow 환경 | 전환 범위 확정 |
| 설계 | 기준, 구조, AI 적용, Airflow 아키텍처 | 검증 및 설계 기준 승인 |
| 개발 | 구현 범위, 예외 처리, DAG 범위 | 오픈 대상 승인 |
| 테스트 | 정합성, 결함, Go/No-Go, DAG 테스트 | 오픈 승인 |
| 이관 | 운영, 교육, 종료, Airflow 인수인계 | 운영 이관 및 종료 승인 |
